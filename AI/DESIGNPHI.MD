# \# Design Philosophy

# 

# 《北行》不是资源模拟器。

# 

# 不是文字AVG。

# 

# 不是小说。

# 

# 玩家真正玩的，

# 

# 不是事件。

# 

# 而是：

# 

# 选择。

# 

# 所有设计都应该围绕：

# 

# "代价"

# 

# 而不是：

# 

# "奖励"

# 

# 玩家不应该思考：

# 

# 哪个奖励最大。

# 

# 而应该思考：

# 

# 我愿意承担什么。

# 

# \---

# 

# 我们相信：

# 

# 失去

# 

# 比获得

# 

# 更容易让玩家记住游戏。

# 

# 因此，

# 

# 尽量设计：

# 

# 不可逆的选择。

# 

# 避免：

# 

# 完美结局。

# 

# 避免：

# 

# 所有资源都能刷回来。

# 

# \---

# 

# 《北行》不是关于世界毁灭。

# 

# 而是关于：

# 

# 人在世界毁灭时，

# 

# 仍然如何活着。
